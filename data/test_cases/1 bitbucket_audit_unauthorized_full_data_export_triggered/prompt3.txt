Generate a SIGMA rule to detect when a user triggers a FULL DATA EXPORT on an
Atlassian Bitbucket instance without proper authorization — a strong indicator
of data exfiltration, insider threat, or compromised admin credentials.

Target product: bitbucket; log source service: audit. Requires the "Advanced"
audit log level enabled on Bitbucket — document this in the rule.

Detection matches Bitbucket audit events using the field names documented in
the Atlassian audit-log-events reference (category + action pattern).

- https://confluence.atlassian.com/bitbucketserver/audit-log-events-776640423.html
- https://confluence.atlassian.com/bitbucketserver/secret-scanning-1157471613.html
