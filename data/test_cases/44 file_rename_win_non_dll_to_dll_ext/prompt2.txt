# Threat to detect
File rename operation where a non-DLL file is renamed to a .dll extension —
a masquerading technique used by malware to evade extension-based detections
after dropping a payload with an innocuous initial extension.

# Target environment
- Product: windows; log source category: file_rename
- Requires Microsoft-Windows-Kernel-File provider with at least the
  KERNEL_FILE_KEYWORD_RENAME_SETLINK_PATH keyword enabled — document this in
  the rule.
- Detection: target filename ends with .dll, but must EXCLUDE many legitimate
  rename patterns:
  (a) source already had .dll extension (legitimate DLL rename);
  (b) source had .tmp extension (installer staging);
  (c) source filename is empty or null;
  (d) the operating process is TiWorker.exe under \Windows\WinSxS\
      (Windows servicing);
  (e) Windows Update activity — wuauclt.exe as image OR target path under
      :\$WINDOWS.~BT\Sources\ (these alternatives form a single filter);
  (f) generic Program Files installers;
  (g) Squirrel auto-updater temporary files (optional filter).
- Use grouped filters: condition must be
  `selection and not 1 of filter_main_* and not 1 of filter_optional_*`.
