Generate a SIGMA rule to detect file rename operations where a non-DLL file
is renamed to .dll — a masquerading technique used by malware to evade
extension-based detections.

Target product: windows; log source category: file_rename. Requires
Microsoft-Windows-Kernel-File provider with KERNEL_FILE_KEYWORD_RENAME_SETLINK_PATH
keyword — document this.

Detection: target filename ends with .dll, but EXCLUDE legitimate patterns
in grouped filters:
- main filters: source already .dll, source .tmp (installers), empty/null
  source, TiWorker.exe under WinSxS, Windows Update (wuauclt.exe OR
  $WINDOWS.~BT\Sources\ path as alternatives), generic Program Files images;
- optional filter: Squirrel auto-updater temp files.

Use condition:
`selection and not 1 of filter_main_* and not 1 of filter_optional_*`.
