Generate a SIGMA rule to detect a new "Allow" rule being added to the Windows
Firewall exception list by the WMI provider host process (WmiPrvSE.EXE).
Ransomware groups such as Rhysida abuse this technique — typically via the
PowerShell cmdlet New-NetFirewallRule or WMI CIM class MSFT_NetFirewallRule —
to open inbound ports for lateral movement, C2, or exfiltration.

Target product: windows; log source service: firewall-as (Windows Firewall
with Advanced Security audit logs).

Detection combines three conditions: (a) the event indicates a rule was ADDED
(include all relevant EventID variants across modern Windows versions);
(b) the rule action is "Allow", not "Block"; (c) the modifying process is
WmiPrvSE.EXE running from its legitimate system path (use a path-suffix match
to avoid trivial bypass).
