# Threat to detect
A new "Allow" rule is added to the Windows Firewall exception list by the
WMI provider host process (WmiPrvSE.EXE). Ransomware groups such as Rhysida
abuse this technique — typically via the PowerShell cmdlet New-NetFirewallRule
or the WMI CIM class MSFT_NetFirewallRule — to open inbound ports for lateral
movement, C2, or exfiltration.

# Target environment
- Product: windows; log source service: firewall-as (Windows Firewall with
  Advanced Security audit logs)
- Detection must combine three conditions:
  (a) the event indicates a rule was ADDED to the firewall (include all
      relevant EventID variants across modern Windows versions);
  (b) the rule action is "Allow", not "Block";
  (c) the modifying process is WmiPrvSE.EXE running from its legitimate
      system path (use a path-suffix match to avoid trivial bypass).
