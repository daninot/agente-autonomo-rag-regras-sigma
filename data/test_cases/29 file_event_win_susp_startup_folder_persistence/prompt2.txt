# Threat to detect
Creation of potentially malicious script or executable files in the Windows
Startup folder — a common persistence technique where files are automatically
executed at user login. Frequently observed in malvertising campaigns and
info-stealer distribution.

# Target environment
- Product: windows; log source category: file_event
- Detection combines two conditions on the same field (all must match):
  (a) file path contains the Windows Startup folder path (|contains);
  (b) file extension matches a list of script and executable types commonly
      abused for persistence: PowerShell, VBScript, JavaScript, batch, HTA,
      DLL, JAR, and similar (|endswith with a comprehensive list).
