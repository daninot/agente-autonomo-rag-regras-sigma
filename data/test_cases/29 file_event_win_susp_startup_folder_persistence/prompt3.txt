Generate a SIGMA rule to detect creation of potentially malicious script or
executable files in the Windows Startup folder — a persistence technique where
files execute automatically at user login, frequently abused by info-stealers
and malvertising campaigns.

Target product: windows; log source category: file_event.

Detection combines |contains for the Startup folder path with |endswith for
a comprehensive list of abusable extensions: PowerShell, VBScript, JavaScript,
batch, HTA, DLL, JAR, and similar script/executable types.
